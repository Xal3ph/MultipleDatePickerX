What is MultipleMonthPicker ?

MultipleMonthPicker is an Angular directive to show a simple calendar allowing user to select multiple dates, a callback is called, you can specify some off days or already selected days.

Get it bower style

bower install multiple-date-picker-x

Get it git style

git clone https://github.com/arca-computing/MultipleMonthPicker.git

Time to use it

Include files

If you are using bower and grunt, you can use wiredep to auto include dependencies in your html file. If you want to include files manually, add those two : 

<script type="text/javascript" src="multipleMonthPicker.min.js"></script>
              <link rel="stylesheet" type="text/css" href="multiple-date-picker-x.css"/>

Add it to your angular app

var app = angular.module('myApp', ['multipleMonthPicker']);

Add it in your html

<div>
  <multiple-date-picker-x/>
</div>

It will use the max width of it's parent, so If you want to reduce it specify a width (or use Bootstrap/Foundation).

<div style="width:50%">
  <multiple-date-picker-x/>
</div>

Add some options

You can give options directly in the html.

Click callback

Option "callback" has been deprecated, please use "month-click" instead

The callback will be called everytime a date is clicked. Be aware that the date given to the callback is a day at midnight.

This exemple shows how to pass a scope function that will prompt an alert with infos.

<multiple-date-picker-x month-click="logInfos"/>

The logInfos function will be called with a preventable event and the moment object:

$scope.logInfos = function(event, date) {
    event.preventDefault() // prevent the select to happen
    console.log(date.valueOf()) //will give you the timestamp
    //reproduce the standard behavior
    date.selected = !date.selected
}

Hover callback

<multiple-date-picker-x month-hover="logInfos"/>

Same options as before

$scope.logInfos = function(event, date) {
    event.preventDefault() // prevent the date.hover to be toggled
    console.log(date.valueOf()) //will give you the timestamp
    if(event.type == 'mouseover') {
        //we're on it!
    }
}
This will be showed to the web console

Month changed callback

The callback will be called everytime you change month (previous or next month). Be aware that newMonth and oldMonth given to the callback is first day of month at midnight.

This exemple shows how to pass a scope function that will prompt an alert with infos.

<multiple-date-picker-x year-changed="logMonthChanged"/>
$scope.logMonthChanged = function(newMonth, oldMonth){
    alert('new month : ' + newMonth.format('YYYY-M-DD') + ' || old month : ' + oldMonth.format('YYYY-M-DD'));
};

All Days-off

Disable all dates in the calendar, default is false

<multiple-date-picker-x all-months-off="true"/>

Highlight-dates

This is an array containing dates you want to highlight, I can be a day-off like a bank day, holiday, your birthday, anything you want.

{
    date: Timestamp/String/Moment, //anything usable with moment
    css: String //class applied, can be "class1" or "class1 class2 class3 ..."
    selectable: true/false,
    title: String //will be use in title attribute (show when cursor is hover)
}

The live demo represent the 2nd of month as holiday (not clickable), 14h a day off (not clickable) and the 25th is my birthday (not really, and clickable).

$scope.highlightDates = [
    {date: moment().date(2).valueOf(), css: 'holiday', selectable: false,title: 'Holiday time !'},
    {date: moment().date(14).valueOf(), css: 'off', selectable: false,title: 'We don\'t work today'},
    {date: moment().date(25).valueOf(), css: 'birthday', selectable: true,title: 'I\'m thir... i\'m 28, seriously, I mean ...'}
];
<multiple-date-picker-x highlight-dates="highlightDates"/>

And here the css used for this demo

/*apply colors if not selected, if selected I want the default selected css*/
.picker-day.holiday:not(.picker-selected){
    background-color: #00ffa2;
    color: #fff;
}

/*if my holiday isn't clickable (it's the case in the demo)*/
.picker-day.holiday.picker-off{
    background-color: #00a267;
    color: #fff;
    cursor: default;
}

/*my birthday, is clickable*/
.picker-day.birthday:not(.picker-selected){
    background-color: #ded100;
    color: #000;
}

Days-off

Option "months-off" is deprecated since 1.2, please use "highlight-dates" instead

Those dates are not selectable (like bank days). It's an array of milliseconds timestamps.

<multiple-date-picker-x months-off="[1404770400000, 1405029600000]"/>

The live demo represent the 14h day off this month with this code sample. oneDayOff is a scope variable.

<multiple-date-picker-x months-off="oneDayOff"/>

Days-selected

Those are pre-selected dates when building the calendar. It's an array of milliseconds timestamps.

<multiple-date-picker-x months-selected="[1404770400000, 1405029600000]"/>

The live demo reprensent the 4th, 5th and 8th day this month with this code sample. selectedDays is a scope variable.

<multiple-date-picker-x months-off="selectedDays"/>

Disallow-back-past-months

If true the back button to go previous month will be disabled if you are on today's month, so you can't go in past month

<multiple-date-picker-x disallow-back-past-years="true"/>

Disallow-go-futur-months

If true the back button to go next month will be disabled if you are on today's month, so you can't go the futur

<multiple-date-picker-x disallow-go-futur-years="true"/>

hide-buttons

If true, all buttons (forward backward) are hidden.

<multiple-date-picker-x hide-buttons="true"/>

count

Add number of count to show multiple calendars.

<multiple-date-picker-x count="6"/>

Let's get serious

You can (of course) use all options at the same time.

<multiple-date-picker-x
            months-off="oneDayOff"
            months-selected="selectedDays"
            month-click="logInfos"/>

What's next ?
